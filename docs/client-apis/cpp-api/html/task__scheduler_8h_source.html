55 // Schedules the dependencies of the given task and finally the task one after another (so
56 // not concurrently), and throws an exception if any of the tasks errors. Regardless of
57 // whether or not the given task or one of its dependencies errors, when this function
58 // returns, no task related to the given task will be in the task queue. Further no worker
59 // thread will be working on the given task.
60 void scheduleTaskAndWaitOrError(const std::shared_ptr<Task>& task);
61
62 // If a user, e.g., currently the copier, adds a set of tasks T1, ..., Tk, to the task scheduler
63 // without waiting for them to finish, the user needs to call waitAllTasksToCompleteOrError() if
64 // it wants to catch the errors that may have happened in T1, ..., Tk. If this function is not
65 // called and a T_i errors, there will be two side effects: (1) the user will never be aware
66 // that there was an error in some of the scheduled tasks; (2) the erroring task will not be
67 // removed from the task queue, so it will remain there permanently. We only remove erroring
68 // tasks inside waitAllTasksToCompleteOrError and scheduleTaskAndWaitOrError. Also, see the note
69 // below in waitAllTasksToCompleteOrError for details of the behavior when multiple tasks fail.
70 std::shared_ptr<ScheduledTask> scheduleTask(const std::shared_ptr<Task>& task);
71
72 // Also note that if a user has scheduled multiple concrete tasks and calls
73 // waitAllTasksToCompleteOrError and multiple tasks error, then waitAllTasksToCompleteOrError
74 // will rethrow only the exception of the first task that it observes to have an error and
75 // it will remove only that one. Other tasks that may have failed many not be removed
76 // from the task queue and remain in the queue. So for now, use this function if you
77 // want the system to crash if any of the tasks fails.
78 void waitAllTasksToCompleteOrError();
