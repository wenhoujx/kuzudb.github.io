62// HashIndex is the entrance to handle all updates and lookups into the index after building from
63// scratch through InMemHashIndex.
64// The index consists of two parts, one is the persistent storage (from the persistent index file),
65// and the other is the local storage. All lookups/deletions/insertions go through local storage,
66// and then the persistent storage if necessary.
67//
68// Key interfaces:
69// - lookup(): Given a key, find its result. Return true if the key is found, else, return false.
70// Lookups go through the local storage first, check if the key is marked as deleted or not, then
71// check whether it can be found inside local insertions or not. If the key is neither marked as
72// deleted nor found in local insertions, we proceed to lookups in the persistent store.
73// - delete(): Delete the given key.
74// Deletions are directly marked in the local storage.
75// - insert(): Insert the given key and value. Return true if the given key doesn't exist in the
76// index before insertion, otherwise, return false.
77// First check if the key to be inserted already exists in local insertions or the persistent
78// store. If the key doesn't exist yet, append it to local insertions, and also remove it from
79// local deletions if it was marked as deleted.
