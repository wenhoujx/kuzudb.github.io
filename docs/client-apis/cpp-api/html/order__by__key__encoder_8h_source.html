29// The OrderByKeyEncoder encodes all columns in the ORDER BY clause into a single binary sequence
30// that, when compared using memcmp will yield the correct overall sorting order. On little-endian
31// hardware, the least-significant byte is stored at the smallest address. To encode the sorting
32// order, we need the big-endian representation for values. For example: we want to encode 73(INT64)
33// and 38(INT64) as an 8-byte binary string The encoding in little-endian hardware is:
34// 73=0x4900000000000000 38=0x2600000000000000, which doesn't preserve the order. The encoding in
35// big-endian hardware is: 73=0x0000000000000049 38=0x0000000000000026, which can easily be compared
36// using memcmp. In addition, The first bit is also flipped to preserve ordering between positive
37// and negative numbers. So the final encoding for 73(INT64) and 38(INT64) as an 8-byte binary
38// string is: 73=0x8000000000000049 38=0x8000000000000026. To handle the null in comparison, we
39// add an extra byte(called the NULL flag) to represent whether this value is null or not.
